\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Char\+Type$>$\newline
struct Generic\+String\+Ref$<$ Char\+Type $>$}

\begin{DoxyTemplParams}{Template Parameters}
{\em Char\+Type} & character type of the string\\
\hline
\end{DoxyTemplParams}
This helper class is used to automatically infer constant string references for string literals, especially from {\ttfamily const} {\bfseries{}}(!) character arrays.

The main use is for creating JSON string values without copying the source string via an \doxysectlink{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2dom_8zh-cn_Allocator}{Allocator}{2}. This requires that the referenced string pointers have a sufficient lifetime, which exceeds the lifetime of the associated \doxylink{class_generic_value}{Generic\+Value}.

{\bfseries{Example}}

\DoxyCodeLine{\mbox{\hyperlink{class_generic_value}{Value}}\ v(\textcolor{stringliteral}{"{}foo"{}});\ \ \ \textcolor{comment}{//\ ok,\ no\ need\ to\ copy\ \&\ calculate\ length}}
\DoxyCodeLine{\textcolor{keyword}{const}\ \textcolor{keywordtype}{char}\ \mbox{\hyperlink{namespacefoo}{foo}}[]\ =\ \textcolor{stringliteral}{"{}foo"{}};}
\DoxyCodeLine{v.SetString(\mbox{\hyperlink{namespacefoo}{foo}});\ \textcolor{comment}{//\ ok}}
\DoxyCodeLine{}

This constructor implicitly creates a constant string reference from a {\ttfamily const} character array. It has better performance than \doxylink{struct_generic_string_ref_aa6b9fd9f6aa49405a574c362ba9af6b5}{String\+Ref(const Char\+Type\texorpdfstring{$\ast$}{*})} by inferring the string \doxylink{struct_generic_string_ref_a4a96d618744ad73f766a1551b1d517fe}{length} from the array length, and also supports strings containing null characters.
