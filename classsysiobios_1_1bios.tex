\doxysubsection{Detailed Description}
The {\ttfamily sysio.\+bios} is the first sample of system contract provided by {\ttfamily block.\+one} through the EOSIO platform. It is a minimalist system contract because it only supplies the actions that are absolutely critical to bootstrap a chain and nothing more. This allows for a chain agnostic approach to bootstrapping a chain.

Just like in the {\ttfamily sysio.\+system} sample contract implementation, there are a few actions which are not implemented at the contract level ({\ttfamily newaccount}, {\ttfamily updateauth}, {\ttfamily deleteauth}, {\ttfamily linkauth}, {\ttfamily unlinkauth}, {\ttfamily canceldelay}, {\ttfamily onerror}, {\ttfamily setabi}, {\ttfamily setcode}), they are just declared in the contract so they will show in the contract\textquotesingle{}s ABI and users will be able to push those actions to the chain via the account holding the {\ttfamily sysio.\+system} contract, but the implementation is at the EOSIO core level. They are referred to as EOSIO native actions.

Link authorization action assigns a specific action from a contract to a permission you have created. Five system actions can not be linked {\ttfamily updateauth}, {\ttfamily deleteauth}, {\ttfamily linkauth}, {\ttfamily unlinkauth}, and {\ttfamily canceldelay}. This is useful because when doing authorization checks, the EOSIO based blockchain starts with the action needed to be authorized (and the contract belonging to), and looks up which permission is needed to pass authorization validation. If a link is set, that permission is used for authoraization validation otherwise then active is the default, with the exception of {\ttfamily \doxylink{namespacefc_a8b03fb95276f683bde05b7715d9f0014}{sysio.\+any}}. {\ttfamily \doxylink{namespacefc_a8b03fb95276f683bde05b7715d9f0014}{sysio.\+any}} is an implicit permission which exists on every account; you can link actions to {\ttfamily \doxylink{namespacefc_a8b03fb95276f683bde05b7715d9f0014}{sysio.\+any}} and that will make it so linked actions are accessible to any permissions defined for the account.
