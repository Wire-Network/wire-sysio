\chapter{Schema}
\hypertarget{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema}{}\label{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema}\index{Schema@{Schema}}
(This feature was released in v1.\+1.\+0)

JSON \doxylink{class_schema}{Schema} is a draft standard for describing the format of JSON data. The schema itself is also JSON data. By validating a JSON structure with JSON \doxylink{class_schema}{Schema}, your code can safely access the DOM without manually checking types, or whether a key exists, etc. It can also ensure that the serialized JSON conform to a specified schema.

Rapid\+JSON implemented a JSON \doxylink{class_schema}{Schema} validator for \href{http://json-schema.org/documentation.html}{\texttt{ JSON Schema Draft v4}}. If you are not familiar with JSON \doxylink{class_schema}{Schema}, you may refer to \href{http://spacetelescope.github.io/understanding-json-schema/}{\texttt{ Understanding JSON Schema}}.\hypertarget{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema_Basic}{}\doxysection{\texorpdfstring{Basic Usage}{Basic Usage}}\label{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema_Basic}
First of all, you need to parse a JSON \doxylink{class_schema}{Schema} into {\ttfamily Document}, and then compile the {\ttfamily Document} into a {\ttfamily Schema\+Document}.

Secondly, construct a {\ttfamily Schema\+Validator} with the {\ttfamily Schema\+Document}. It is similar to a {\ttfamily \doxylink{class_writer}{Writer}} in the sense of handling SAX events. So, you can use {\ttfamily document.\+Accept(validator)} to validate a document, and then check the validity.

Unlike most JSON \doxylink{class_schema}{Schema} validator implementations, Rapid\+JSON provides a SAX-\/based schema validator. Therefore, you can parse a JSON from a stream while validating it on the fly. If the validator encounters a JSON value that invalidates the supplied schema, the parsing will be terminated immediately. This design is especially useful for parsing large JSON files.\hypertarget{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema_DOM}{}\doxysubsection{\texorpdfstring{DOM parsing}{DOM parsing}}\label{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema_DOM}

Rapid\+JSON passed 262 out of 263 tests in \href{https://github.com/json-schema/JSON-Schema-Test-Suite}{\texttt{ JSON Schema Test Suite}} (Json \doxylink{class_schema}{Schema} draft 4).

The failed test is "{}changed scope ref invalid"{} of "{}change resolution scope"{} in {\ttfamily ref\+Remote.\+json}. It is due to that {\ttfamily id} schema keyword and URI combining function are not implemented.

And 1400x faster than the slowest one.\hypertarget{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema_Reporting}{}\doxysection{\texorpdfstring{Schema violation reporting}{Schema violation reporting}}\label{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema_Reporting}
(Unreleased as of 2017-\/09-\/20)

When validating an instance against a JSON \doxylink{class_schema}{Schema}, it is often desirable to report not only whether the instance is valid, but also the ways in which it violates the schema.

The {\ttfamily Schema\+Validator} class collects errors encountered during validation into a JSON {\ttfamily Value}. This error object can then be accessed as {\ttfamily validator.\+Get\+Error()}.

The structure of the error object is subject to change in future versions of Rapid\+JSON, as there is no standard schema for violations. The details below this point are provisional only.\hypertarget{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema_ReportingGeneral}{}\doxysubsection{\texorpdfstring{General provisions}{General provisions}}\label{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2schema_ReportingGeneral}
Validation of an instance value against a schema produces an error value. The error value is always an object. An empty object {\ttfamily \{\}} indicates the instance is valid.


\begin{DoxyItemize}
\item The name of each member corresponds to the JSON \doxylink{class_schema}{Schema} keyword that is violated.
\item The value is either an object describing a single violation, or an array of such objects.
\end{DoxyItemize}

Each violation object contains two string-\/valued members named {\ttfamily instance\+Ref} and {\ttfamily schema\+Ref}. {\ttfamily instance\+Ref} contains the URI fragment serialization of a JSON Pointer to the instance subobject in which the violation was detected. {\ttfamily schema\+Ref} contains the URI of the schema and the fragment serialization of a JSON Pointer to the subschema that was violated.

Individual violation objects can contain other keyword-\/specific members. These are detailed further.
