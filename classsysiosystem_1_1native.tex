void sysiosystem\+::native\+::linkauth (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{structsysio_1_1chain_1_1name}{name}}}]{account}{, }\item[{\mbox{\hyperlink{structsysio_1_1chain_1_1name}{name}}}]{code}{, }\item[{\mbox{\hyperlink{structsysio_1_1chain_1_1name}{name}}}]{type}{, }\item[{\mbox{\hyperlink{structsysio_1_1chain_1_1name}{name}}}]{requirement}{, }\item[{binary\+\_\+extension$<$ \mbox{\hyperlink{structsysio_1_1chain_1_1name}{name}} $>$}]{authorized\+\_\+by}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}

Link authorization action assigns a specific action from a contract to a permission you have created. Five system actions can not be linked {\ttfamily updateauth}, {\ttfamily deleteauth}, {\ttfamily linkauth}, {\ttfamily unlinkauth}, and {\ttfamily canceldelay}. This is useful because when doing authorization checks, the EOSIO based blockchain starts with the action needed to be authorized (and the contract belonging to), and looks up which permission is needed to pass authorization validation. If a link is set, that permission is used for authorization validation otherwise then active is the default, with the exception of {\ttfamily \doxylink{namespacefc_a8b03fb95276f683bde05b7715d9f0014}{sysio.\+any}}. {\ttfamily \doxylink{namespacefc_a8b03fb95276f683bde05b7715d9f0014}{sysio.\+any}} is an implicit permission which exists on every account; you can link actions to {\ttfamily \doxylink{namespacefc_a8b03fb95276f683bde05b7715d9f0014}{sysio.\+any}} and that will make it so linked actions are accessible to any permissions defined for the account.

This contract enforces additional rules\+:


\begin{DoxyEnumerate}
\item If authorized\+\_\+by is present and not "{}"{}, then the contract does a require\+\_\+auth2(account, authorized\+\_\+by).
\item If the account has opted into limitauthchg, then authorized\+\_\+by must be present and not "{}"{}.
\item If the account has opted into limitauthchg, and allow\+\_\+perms is not empty, then authorized\+\_\+by must be in the array.
\item If the account has opted into limitauthchg, and disallow\+\_\+perms is not empty, then authorized\+\_\+by must not be in the array.
\end{DoxyEnumerate}

These actions map one-\/on-\/one with the ones defined in core layer of EOSIO, that\textquotesingle{}s where their implementation actually is done. They are present here only so they can show up in the abi file and thus user can send them to this contract, but they have no specific implementation at this contract level, they will execute the implementation at the core layer and nothing else. New account action is called after a new account is created. This code enforces resource-\/limits rules for new accounts as well as new account naming conventions.

Account names containing \textquotesingle{}.\textquotesingle{} symbols must have a suffix equal to the name of the creator. This allows users who buy a premium name (shorter than 12 characters with no dots) to be the only ones who can create accounts with the creator\textquotesingle{}s name as a suffix.
