The iostream transport does not provide any security features. As such it defaults to returning false when {\ttfamily is\+\_\+secure} is called. However, the iostream transport may be used to wrap an external socket API that may provide secure transport. This method allows that external API to flag whether or not it can create secure connections so that users of the Web\+Socket++ API will get more accurate information.

Setting this value only indicates whether or not the endpoint is capable of producing and managing secure connections. Connections produced by this endpoint must also be individually flagged as secure if they are.
