\doxysubsection{Detailed Description}
The {\ttfamily sysio.\+system} smart contract is provided by {\ttfamily block.\+one} as a sample system contract, and it defines the structures and actions needed for blockchain\textquotesingle{}s core functionality.

Just like in the {\ttfamily sysio.\+bios} sample contract implementation, there are a few actions which are not implemented at the contract level ({\ttfamily newaccount}, {\ttfamily updateauth}, {\ttfamily deleteauth}, {\ttfamily linkauth}, {\ttfamily unlinkauth}, {\ttfamily canceldelay}, {\ttfamily onerror}, {\ttfamily setabi}, {\ttfamily setcode}), they are just declared in the contract so they will show in the contract\textquotesingle{}s ABI and users will be able to push those actions to the chain via the account holding the {\ttfamily sysio.\+system} contract, but the implementation is at the EOSIO core level. They are referred to as EOSIO native actions.


\begin{DoxyItemize}
\item Users can stake tokens for CPU and Network bandwidth, and then vote for producers or delegate their vote to a proxy.
\item Producers register in order to be voted for, and can claim per-\/block and per-\/vote rewards.
\item Users can buy and sell RAM at a market-\/determined price.
\item Users can bid on premium names.
\item \doxylink{struct_a}{A} resource exchange system (REX) allows token holders to lend their tokens, and users to rent CPU and Network resources in return for a market-\/determined fee. 
\end{DoxyItemize}
