typedef int(\texorpdfstring{$\ast$}{*} secp256k1\+\_\+nonce\+\_\+function\+\_\+hardened) (unsigned char \texorpdfstring{$\ast$}{*}nonce32, const unsigned char \texorpdfstring{$\ast$}{*}msg, size\+\_\+t msglen, const unsigned char \texorpdfstring{$\ast$}{*}key32, const unsigned char \texorpdfstring{$\ast$}{*}xonly\+\_\+pk32, const unsigned char \texorpdfstring{$\ast$}{*}algo, size\+\_\+t algolen, void \texorpdfstring{$\ast$}{*}data)}

This module implements a variant of Schnorr signatures compliant with Bitcoin Improvement Proposal 340 "{}\+Schnorr Signatures for secp256k1"{} (\href{https://github.com/bitcoin/bips/blob/master/bip-0340.mediawiki}{\texttt{ https\+://github.\+com/bitcoin/bips/blob/master/bip-\/0340.\+mediawiki}}). \doxylink{struct_a}{A} pointer to a function to deterministically generate a nonce.

Same as secp256k1\+\_\+nonce function with the exception of accepting an additional pubkey argument and not requiring an attempt argument. The pubkey argument can protect signature schemes with key-\/prefixed challenge hash inputs against reusing the nonce when signing with the wrong precomputed pubkey.

Returns\+: 1 if a nonce was successfully generated. 0 will cause signing to return an error. Out\+: nonce32\+: pointer to a 32-\/byte array to be filled by the function In\+: msg\+: the message being verified. Is NULL if and only if msglen is 0. msglen\+: the length of the message key32\+: pointer to a 32-\/byte secret key (will not be NULL) xonly\+\_\+pk32\+: the 32-\/byte serialized xonly pubkey corresponding to key32 (will not be NULL) algo\+: pointer to an array describing the signature algorithm (will not be NULL) algolen\+: the length of the algo array data\+: arbitrary data pointer that is passed through

Except for test cases, this function should compute some cryptographic hash of the message, the key, the pubkey, the algorithm description, and data.

Create a Schnorr signature.

Does {\itshape not} strictly follow BIP-\/340 because it does not verify the resulting signature. Instead, you can manually use secp256k1\+\_\+schnorrsig\+\_\+verify and abort if it fails.

This function only signs 32-\/byte messages. If you have messages of a different size (or the same size but without a context-\/specific tag prefix), it is recommended to create a 32-\/byte message hash with secp256k1\+\_\+tagged\+\_\+sha256 and then sign the hash. Tagged hashing allows providing an context-\/specific tag for domain separation. This prevents signatures from being valid in multiple contexts by accident.

Returns 1 on success, 0 on failure. Args\+: ctx\+: pointer to a context object, initialized for signing. Out\+: sig64\+: pointer to a 64-\/byte array to store the serialized signature. In\+: msg32\+: the 32-\/byte message being signed. keypair\+: pointer to an initialized keypair. aux\+\_\+rand32\+: 32 bytes of fresh randomness. While recommended to provide this, it is only supplemental to security and can be NULL. \doxylink{struct_a}{A} NULL argument is treated the same as an all-\/zero one. See BIP-\/340 "{}\+Default Signing"{} for a full explanation of this argument and for guidance if randomness is expensive.

\mbox{\hyperlink{secp256k1_8h_a3f1967acd0fe26283877c61b3f2694a5}{SECP256\+K1\+\_\+\+API}} const \mbox{\hyperlink{secp256k1__schnorrsig_8h_a134d980bb4432c6d70e3298979c8f5ec}{secp256k1\+\_\+nonce\+\_\+function\+\_\+hardened}} secp256k1\+\_\+nonce\+\_\+function\+\_\+bip340\hspace{0.3cm}{\ttfamily [extern]}}

An implementation of the nonce generation function as defined in Bitcoin Improvement Proposal 340 "{}\+Schnorr Signatures for secp256k1"{} (\href{https://github.com/bitcoin/bips/blob/master/bip-0340.mediawiki}{\texttt{ https\+://github.\+com/bitcoin/bips/blob/master/bip-\/0340.\+mediawiki}}).

If a data pointer is passed, it is assumed to be a pointer to 32 bytes of auxiliary random data as defined in BIP-\/340. If the data pointer is NULL, the nonce derivation procedure follows BIP-\/340 by setting the auxiliary random data to zero. The algo argument must be non-\/\+NULL, otherwise the function will fail and return 0. The hash will be tagged with algo. Therefore, to create BIP-\/340 compliant signatures, algo must be set to "{}\+BIP0340/nonce"{} and algolen to 13.
