\doxysubsection{Detailed Description}
This class records the links from contracts and message types within those contracts to permission\+\_\+objects defined by the user to record the authority required to execute those messages within those contracts. For example, suppose we have a contract called "{}currency"{} and that contract defines a message called "{}transfer"{}. Furthermore, suppose a user, "{}joe"{}, has a permission level called "{}money"{} and joe would like to require that permission level in order for his account to invoke currency.\+transfer. To do this, joe would create a \doxylink{classsysio_1_1chain_1_1permission__link__object}{permission\+\_\+link\+\_\+object} for his account with "{}currency"{} as the code account, "{}transfer"{} as the message\+\_\+type, and "{}money"{} as the required\+\_\+permission. After this, in order to validate, any message to "{}currency"{} of type "{}transfer"{} that requires joe\textquotesingle{}s approval would require signatures sufficient to satisfy joe\textquotesingle{}s "{}money"{} authority.

Accounts may set links to individual message types, or may set default permission requirements for all messages to a given contract. To set the default for all messages to a given contract, set message\+\_\+type to the empty string. When looking up which permission to use, if a link is found for a particular \{account, code, message\+\_\+type\} triplet, then the required\+\_\+permission from that link is used. If no such link is found, but a link is found for \{account, code, "{}"{}\}, then the required\+\_\+permission from that link is used. If no such link is found, account\textquotesingle{}s active authority is used.
