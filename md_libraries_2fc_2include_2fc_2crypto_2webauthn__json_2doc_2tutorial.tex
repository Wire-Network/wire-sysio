This tutorial introduces the basics of the Document Object Model(\+DOM) API.

As shown in \doxysectlink{index}{Usage at a glance}{0}, JSON can be parsed into a DOM, and then the DOM can be queried and modified easily, and finally be converted back to JSON.\hypertarget{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2tutorial_8zh-cn_ValueDocument}{}\doxysection{\texorpdfstring{Value \& Document}{Value \& Document}}\label{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2tutorial_8zh-cn_ValueDocument}

Note that, an integer value may be obtained in various ways without conversion. For example, \doxylink{struct_a}{A} value {\ttfamily x} containing 123 will make {\ttfamily x.\+Is\+Int() == x.\+Is\+Uint() == x.\+Is\+Int64() == x.\+Is\+Uint64() == true}. But a value {\ttfamily y} containing -\/3000000000 will only make {\ttfamily x.\+Is\+Int64() == true}.

When obtaining the numeric values, {\ttfamily Get\+Double()} will convert internal integer representation to a {\ttfamily double}. Note that, {\ttfamily int} and {\ttfamily unsigned} can be safely converted to {\ttfamily double}, but {\ttfamily int64\+\_\+t} and {\ttfamily uint64\+\_\+t} may lose precision (since mantissa of {\ttfamily double} is only 52-\/bits).\hypertarget{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2tutorial_8zh-cn_QueryString}{}\doxysubsection{\texorpdfstring{Query String}{Query String}}\label{md_libraries_2fc_2include_2fc_2crypto_2webauthn__json_2doc_2tutorial_8zh-cn_QueryString}
In addition to {\ttfamily Get\+String()}, the {\ttfamily Value} class also contains {\ttfamily Get\+String\+Length()}. Here explains why.

According to RFC 4627, JSON strings can contain Unicode character {\ttfamily U+0000}, which must be escaped as {\ttfamily "{}\textbackslash{}u0000"{}}. The problem is that, C/\+C++ often uses null-\/terminated string, which treats \`{}`\textbackslash{}0'\`{} as the terminator symbol.

To conform RFC 4627, Rapid\+JSON supports string containing {\ttfamily U+0000}. If you need to handle this, you can use {\ttfamily Get\+String\+Length()} to obtain the correct string length.

For example, after parsing a the following JSON to {\ttfamily Document d}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{\ "{}s"{}\ :\ \ "{}a\(\backslash\)u0000b"{}\ \}}

\end{DoxyCode}
 The correct length of the value {\ttfamily "{}a\textbackslash{}u0000b"{}} is 3. But {\ttfamily strlen()} returns 1.

{\ttfamily Get\+String\+Length()} can also improve performance, as user may often need to call {\ttfamily strlen()} for allocating buffer.

Besides, {\ttfamily std\+::string} also support a constructor\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{string(\textcolor{keyword}{const}\ \textcolor{keywordtype}{char}*\ \mbox{\hyperlink{yubihsm__pkcs11_8c_a506da415c9307c897ca9be93acfe1d0f}{s}},\ \textcolor{keywordtype}{size\_t}\ \mbox{\hyperlink{gmock__stress__test_8cc_afd9db40e3361ae09188795e8cbe19752}{count}});}

which accepts the length of string as parameter. This constructor supports storing null character within the string, and should also provide better performance.

Why? What is the advantage of this semantics?

The simple answer is performance. For fixed size JSON types (Number, True, False, Null), copying them is fast and easy. However, For variable size JSON types (String, Array, Object), copying them will incur a lot of overheads. And these overheads are often unnoticed. Especially when we need to create temporary object, copy it to another variable, and then destruct it.

For example, if normal {\itshape copy} semantics was used\+:

To make Rapid\+JSON simple and fast, we chose to use {\itshape move} semantics for assignment. It is similar to {\ttfamily std\+::auto\+\_\+ptr} which transfer ownership during assignment. Move is much faster and simpler, it just destructs the original value, {\ttfamily \doxylink{yubihsm__pkcs11_8c_ae785d5badbf1f08b491af31f32a135a1}{memcpy()}} the source to destination, and finally sets the source as Null type.

Rapid\+JSON provides two strategies for storing string.


\begin{DoxyEnumerate}
\item copy-\/string\+: allocates a buffer, and then copy the source data into it.
\item const-\/string\+: simply store a pointer of string.
\end{DoxyEnumerate}

Copy-\/string is always safe because it owns a copy of the data. Const-\/string can be used for storing a string literal, and for in-\/situ parsing which will be mentioned in the DOM section.

To make memory allocation customizable, Rapid\+JSON requires users to pass an instance of allocator, whenever an operation may require allocation. This design is needed to prevent storing a allocator (or Document) pointer per Value.

Therefore, when we assign a copy-\/string, we call this overloaded {\ttfamily Set\+String()} with allocator\+:
