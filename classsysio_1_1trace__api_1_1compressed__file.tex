\doxysubsection{Detailed Description}
wrapper for read-\/only access to a compressed file. compressed files support seeking and reading

the efficiency of seeking is lower than that of an uncompressed file as each seek translates to
\begin{DoxyItemize}
\item 2 seeks + 1 read to load and process the seek-\/point-\/mapping
\item potentially a read/decompress/discard of the data between the seek point and the requested offset
\end{DoxyItemize}

More seek points can lower the average amount of data that must be read/decompressed/discarded in order to seek to any given offset. However, each seek point has some effect on the file size as it represents a flush of the compressor which can degrade compression performance.

\doxylink{struct_a}{A} compressed file looks like this on the filesystem\+: /====================\textbackslash{} file offset 0 \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} Compressed \doxylink{struct_data}{Data} \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} with seek points \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|}-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/---\texorpdfstring{$\vert$}{|} file offset END -\/ 2 -\/ (16 \texorpdfstring{$\ast$}{*} seek point count) \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} mapping of \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} orig offset to \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} seek pt offset \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|}-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/---\texorpdfstring{$\vert$}{|} file offset END -\/ 2 \texorpdfstring{$\vert$}{|} seek pt count \texorpdfstring{$\vert$}{|} ====================/ file offset END

Where a "{}seek point"{} is a point in the compressed data stream where the decompressor can start reading from having not read any of the prior data seek points should be traversable by a decompressor so that reads which span seek points do not have to be aware of them

In zlib this is created by doing a complete flush of the stream
